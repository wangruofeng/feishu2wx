# 快速启动指南

> 💡 **提示**：本项目的核心排版功能为纯前端，无需后端。但「推送到草稿箱」功能需要运行后端服务。

## 方式一：前后端同时启动（推荐）

用于完整体验，包括「推送到草稿箱」。

### 1. 安装依赖

```bash
npm install
```

### 2. 启动服务

```bash
npm run dev
```

前端将在 `http://localhost:3100` 运行，后端将在 `http://localhost:3101` 运行。

## 方式二：仅启动前端

只使用排版、预览、复制功能时可以运行：

```bash
npm start
```

前端默认运行在 `http://localhost:3000`。此模式不会启动后端，推送草稿箱功能不可用。

## 验证安装

1. 打开浏览器访问 `http://localhost:3100`
2. 应该能看到应用界面，包括：
   - 顶部栏：品牌标识、GitHub 链接、主题切换（经典/橙色/蓝色/青绿 4 种）、设置（齿轮图标，弹出排版设置面板）、编辑/预览切换、复制、导出、推送按钮
   - 左侧 Markdown 编辑器：顶部格式工具栏（H1/H2/H3、粗体/斜体/行内代码、列表/引用、链接/图片），底部文件操作（导入、加载示例、清空、字数统计、大纲）
   - 右侧实时预览区域（可切换桌面/手机宽度、全屏预览）
   - 排版设置面板（点击齿轮）：按通用 / 标题 / 正文 / 引用块 / 图片 / 代码块 / 模板 / 公众号 分组管理排版选项
3. 尝试粘贴一些内容到左侧编辑器
4. 右侧应该实时显示预览效果
5. 在设置面板中切换主题配色、字体、代码块样式（极简/现代）、图片样式（默认/边框/阴影）等，查看效果变化
6. 测试代码块语法高亮功能（输入代码块，如 \`\`\`javascript ... \`\`\`）
7. 测试编辑器源码语法高亮（在设置面板「通用」组切换 GitHub/Dracula/Monokai 配色）
8. 点击顶部"复制"测试复制功能（分别在选中部分内容和未选中内容两种情况下）
9. 测试全屏预览功能（点击预览区全屏按钮，按 ESC 退出）

## 常见问题

### 端口被占用

如果 3000 端口被占用，可以：
编辑 `package.json`，在 `scripts.start` 中添加：
```json
"start": "PORT=3001 react-scripts start"
```

### 依赖安装失败

尝试清除缓存后重新安装：
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### 字体加载失败

- 确保网络连接正常（Google Fonts 需要联网）
- 如果网络受限，可以使用系统默认字体
- 某些字体可能需要等待几秒钟才能加载完成

### 全屏预览无法退出

- 按 ESC 键可以退出全屏预览
- 或点击顶部的"退出全屏"按钮
