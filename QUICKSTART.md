# 快速启动指南

> 💡 **提示**：本项目的核心排版功能为纯前端，无需后端。但「推送到草稿箱」功能需要运行后端服务。

## 方式一：前后端同时启动（推荐）

用于完整体验，包括「推送到草稿箱」。

### 1. 安装依赖

```bash
npm install
```

### 2. 启动服务

```bash
npm run dev
```

前端将在 `http://localhost:3100` 运行，后端将在 `http://localhost:3101` 运行。

## 方式二：仅启动前端

只使用排版、预览、复制功能时可以运行：

```bash
npm start
```

前端默认运行在 `http://localhost:3000`。此模式不会启动后端，推送草稿箱功能不可用。

## 验证安装

1. 打开浏览器访问 `http://localhost:3100`
2. 应该能看到应用界面，包括：
   - 顶部标题（"飞书文档"为飞书主题色 #00BECA，"微信公众号"为微信主题色 #07C160）
   - 第一行工具栏：字体选择、设备预览切换、隐藏源码、全屏预览、设置面板（齿轮图标）
   - 第二行工具栏：主题切换（4 种主题横向排列）
   - 左侧 Markdown 编辑器（可隐藏，支持快捷键 B/I/K/Z）
   - 右侧实时预览区域
   - 底部工具栏：加载示例、清空、H1 底线切换、H1 反显切换、分割线切换、图片样式切换、代码块样式切换、一键复制公众号
3. 尝试粘贴一些内容到左侧编辑器
4. 右侧应该实时显示预览效果
5. 尝试切换主题和字体，查看效果变化
6. 测试代码块语法高亮功能（输入代码块，如 \`\`\`javascript ... \`\`\`），并在底部工具栏中切换经典/现代代码块样式
7. 测试行内代码加粗效果（输入 \`code\`）
8. 测试 H1 底线切换功能（点击"隐藏 H1 底线"按钮）
9. 测试图片样式切换功能（点击"边框模式"或"阴影模式"按钮）
10. 点击"一键复制公众号"测试复制功能（分别在选中部分内容和未选中内容两种情况下）
11. 测试全屏预览功能（点击全屏按钮，按 ESC 退出）

## 常见问题

### 端口被占用

如果 3000 端口被占用，可以：
编辑 `package.json`，在 `scripts.start` 中添加：
```json
"start": "PORT=3001 react-scripts start"
```

### 依赖安装失败

尝试清除缓存后重新安装：
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

### 字体加载失败

- 确保网络连接正常（Google Fonts 需要联网）
- 如果网络受限，可以使用系统默认字体
- 某些字体可能需要等待几秒钟才能加载完成

### 全屏预览无法退出

- 按 ESC 键可以退出全屏预览
- 或点击顶部的"退出全屏"按钮
